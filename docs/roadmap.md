# Development Roadmap

This document outlines the planned development phases for the Competitor Analysis workflow.

---

## 🗺️ Phases

### 🟩 Phase 1: Project Foundation & Workflow (Current)
* Setup the directory skeleton, core schemas, configuration details, and the main workflow orchestrator.

### ⬜ Phase 2: Competitor Discovery Skill
* Implement search automation and domain scraping using the `fetcher` and `playwright` MCPs.
* Dynamically generate primary competitor list.

### ⬜ Phase 3: Competitor Analysis & Scraping
* Implement pricing model analysis, feature matrix mapping, and qualitative analysis.

### ⬜ Phase 4: UX Pattern Extraction & Screenshot Capture
* Automate flow navigation using Playwright, capture visual screenshot assets, and map interaction sequences.

### ⬜ Phase 5: Opportunity & Gap Analysis
* Build logic to identify feature gaps and design system parities.

### ⬜ Phase 6: Report Compilation
* Automate populating templates, compiling final reports, and generating stakeholder-ready PDF/Markdown files.

### ⬜ Phase 7: Interactive Refinement
* Add interactive checkpoints so the user can refine discovered competitors or captured screenshots during execution.

### ⬜ Phase 8: Figma Design System Parity
* Deepen the `figma-console` integration to audit design system compliance.

### ⬜ Phase 9: Multi-market Customization
* Customize reports based on target region markets and user segment preferences.

### ⬜ Phase 10: Extensibility & Contribution Guidelines
* Open source packaging and instructions for developer contributions.
